# Security Policy

## Reporting Vulnerabilities

If you discover a security issue, please report it privately to the repository owner.

Include:

- Affected area (frontend/backend/dependency)
- Reproduction steps
- Potential impact
- Suggested mitigation (if available)

## Scope

This application is local-first and intended for personal desktop usage, but standard secure development practices still apply:

- Validate file system inputs
- Keep dependencies current
- Restrict command execution paths
- Review external process interactions (e.g., ffmpeg integrations)
